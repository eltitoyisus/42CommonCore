# CPP Modules 00-04

## Description

The CPP modules (00-04) introduce you to Object-Oriented Programming using C++98. These modules cover fundamental C++ concepts, from basic syntax to more advanced OOP principles.

## Project Overview

Through these five modules, you'll learn:
- C++ syntax and basics
- Object-Oriented Programming principles
- Classes and objects
- Memory management in C++
- Operator overloading
- Inheritance and polymorphism
- Abstract classes and interfaces

## Module Structure

Each module contains multiple exercises (ex00, ex01, ex02, etc.) that progressively introduce new concepts. You must complete exercises in order.

### The Orthodox Canonical Form (OCF)
Every class should have:
1. Default constructor
2. Copy constructor
3. Copy assignment operator
4. Destructor

```cpp
class MyClass {
public:
    MyClass();                              // Default constructor
    MyClass(const MyClass& other);          // Copy constructor
    MyClass& operator=(const MyClass& rhs); // Copy assignment operator
    ~MyClass();                             // Destructor
};
```

- Canonical form implementation

```cpp

protected:
    std::string type;
public:
    virtual void makeSound() const = 0;  // Pure virtual
    virtual ~AAnimal() {}
};

public:
    void makeSound() const { /* Implementation */ }
};
```

## Common Requirements (All Modules)

### Compilation
```bash
c++ -Wall -Wextra -Werror -std=c++98 *.cpp -o program
```

### Coding Standards
- Use C++98 standard
- No C functions (except write, malloc-related in specific cases)
- No using namespace std; in header files
- Each class in separate files (ClassName.hpp, ClassName.cpp)
- Orthodox Canonical Form for all classes (from Module 02+)
- No memory leaks
- Compile with -Wall -Wextra -Werror

### File Naming
```
ClassName.hpp       // Header file
ClassName.cpp       // Implementation file
main.cpp           // Test file
Makefile           // Compilation rules
```

### Forbidden
- STL containers (vector, list, map, etc.) - except in later modules
- `printf` and other C functions
- `using namespace std;` in headers
- Memory leaks
